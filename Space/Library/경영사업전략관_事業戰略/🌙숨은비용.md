# 🌙 숨은비용 (Hidden Costs)

> "보이는 비용은 관리되고, 보이지 않는 비용은 축적된다."

![Hidden Cost Iceberg](hidden_cost_iceberg.png)

---

## 개요

숨은비용은 예산표에 나타나지 않지만 실제로 조직의 효율성, 속도, 품질을 잠식하는 비용이다. 대부분 "당연히 필요한 것"으로 인식되어 측정되지 않는다.

```
┌─────────────────────────────────────────────────────────────────┐
│  보이는 비용 (Visible)     │  숨은 비용 (Hidden)                 │
├─────────────────────────────────────────────────────────────────┤
│  인건비, 재료비, 임대료    │  대기, 재작업, 맥락 손실, 조율     │
│  측정됨, 관리됨            │  측정 안 됨, 축적됨                │
│  예산의 30%                │  실제 비용의 70%?                  │
└─────────────────────────────────────────────────────────────────┘
```

---

## 1. 핸드오프 비용 (Handoff Cost)

### 정의

> 한 사람/팀에서 다른 사람/팀으로 작업을 "넘길 때" 발생하는 비용

### 구성 요소

| 비용 유형 | 설명 | 예시 |
|:---|:---|:---|
| **커뮤니케이션** | 설명, 질문, 응답 시간 | "이 코드는 이렇게 돌려야 해" |
| **맥락 손실** | 전달자의 의도를 수신자가 100% 이해 못함 | 암묵지 → 형식지 변환 손실 |
| **대기 시간** | 상대방이 끝날 때까지 대기 | 병목, 유휴 시간 |
| **재작업** | 오해로 인한 다시 하기 | "아, 그게 아니라..." |
| **품질 저하** | 전체 그림을 못 보는 분업자 | 부분 최적화 |

### 측정 공식

```
핸드오프 비용 = (전달 시간 + 대기 시간 + 오해 수정 시간) × 핸드오프 횟수
```

### 해결책

- **Single-Piece Flow**: 한 사람이 처음부터 끝까지
- **Co-location**: 물리적/가상 근접 배치
- **표준화된 인터페이스**: 전달 형식 통일

### 사례: 4일 Sprint

```
기존: G 설계 → [핸드오프] → J 실행 → [핸드오프] → K 검증
      = 2회 핸드오프 × 7 task × 1시간 = 14시간 손실

개선: G 설계+실행 → [핸드오프] → K 검증
      = 1회 핸드오프 × 7 task × 0.5시간 = 3.5시간 손실

절약: 10.5시간 (75% 감소)
```

## 2. 컨텍스트 스위칭 비용 (Context Switching Cost)

### 정의

> 한 작업에서 다른 작업으로 전환할 때 발생하는 인지적 비용

### 연구 결과

- 전환 후 원래 집중력 회복까지 **평균 23분** 소요 (UC Irvine)
- 멀티태스킹은 IQ를 **15포인트** 낮춤 (London 대학)

### 구성 요소

| 비용 | 설명 |
|:---|:---|
| **Warm-up 시간** | "어디까지 했더라?" 파악 |
| **Working Memory 손실** | 머릿속에 로드된 정보 증발 |
| **오류 증가** | 전환 직후 실수율 50% 증가 |
| **에너지 소모** | 정신적 피로 누적 |

### 측정 공식

```
실제 생산 시간 = 명목 시간 × (1 - 0.2 × 동시 프로젝트 수)

예: 5개 프로젝트 동시 진행
   = 8시간 × (1 - 0.2 × 5) = 8 × 0 = 0시간 (!)
```

### 해결책

- **시간 블로킹**: 한 작업에 2시간 이상 연속 배정
- **WIP 제한**: 동시 진행 작업 수 제한 (이상적: 1~2개)
- **배치 처리**: 유사 작업 모아서 처리

## 3. 조율 비용 (Coordination Cost)

### 정의

> 여러 사람이 함께 일할 때 "맞추기 위해" 드는 비용

### Brooks's Law

> "지연된 소프트웨어 프로젝트에 인력을 추가하면 더 늦어진다"

```
조율 비용 = n(n-1)/2  (n = 팀원 수)

2명: 1개 연결
3명: 3개 연결
4명: 6개 연결
10명: 45개 연결
```

### 구성 요소

| 비용 | 설명 |
|:---|:---|
| **회의** | 정보 동기화, 의사결정 |
| **문서화** | 다른 사람이 이해할 수 있게 기록 |
| **갈등 해결** | 의견 충돌 조정 |
| **대기** | 다른 사람의 결정/결과물 기다림 |

### 측정

```
팀 효율성 = 개인 효율성 × (1 / √n)

4명 팀: 개인의 50% 효율
9명 팀: 개인의 33% 효율
```

### 해결책

- **작은 팀**: 2-Pizza Rule (피자 2판으로 먹일 수 있는 인원)
- **명확한 역할**: 중복 없는 책임 배분
- **비동기 소통**: 불필요한 실시간 회의 제거

---

## 4. 재작업 비용 (Rework Cost)

### 정의

> 처음부터 제대로 하지 않아서 다시 해야 하는 비용

### 1-10-100 Rule

```
발견 시점별 수정 비용:
- 설계 단계: 1
- 개발 단계: 10
- 출시 후: 100
```

### 구성 요소

| 비용 | 설명 |
|:---|:---|
| **원래 작업 시간** | 처음 만든 시간 (손실) |
| **수정 시간** | 고치는 시간 |
| **검증 시간** | 고친 게 맞는지 확인 |
| **기회비용** | 그 시간에 했을 수 있는 다른 일 |

### 측정

```
재작업 비용 = 원래 비용 × 2.5 (평균)

"한 번 고치면 처음 만든 것의 2.5배 비용"
```

### 해결책

- **First Time Right**: 처음부터 제대로
- **빠른 피드백 루프**: 오류 조기 발견
- **체크리스트**: 실수 방지 시스템

---

## 5. 기술 부채 비용 (Technical Debt Cost)

### 정의

> "나중에 고치자"고 한 임시방편의 누적 비용

### 이자율

```
기술 부채 이자 = 원금 × 월 20~30%

6개월 방치: 원금의 3~4배로 증가
```

### 구성 요소

| 비용 | 설명 |
|:---|:---|
| **Workaround 시간** | 문제 피해가는 시간 |
| **인지 부하** | "여기는 건드리면 안 돼" 기억 |
| **신규 인력 온보딩** | 레거시 설명 시간 |
| **대규모 리팩토링** | 결국 해야 하는 정리 |

### 해결책

- **20% Rule**: 매 스프린트 20%를 부채 상환에 투자
- **Boy Scout Rule**: 코드를 발견했을 때보다 깨끗하게 남기기
- **부채 가시화**: 기술 부채를 백로그에 명시

## 6. 대기 비용 (Waiting Cost)

### 정의

> 작업이 "진행 중"이지만 실제로는 멈춰있는 시간의 비용

### Little's Law

```
평균 대기 시간 = WIP / 처리량

WIP(진행 중 작업)이 많을수록 대기 시간 증가
```

### 유형

| 대기 유형 | 예시 |
|:---|:---|
| **승인 대기** | 결재, 리뷰 기다림 |
| **자원 대기** | 장비, 환경, 데이터 |
| **정보 대기** | 질문 답변, 명세 확정 |
| **의존성 대기** | 선행 작업 완료 |

### 측정

```
흐름 효율성 = 실제 작업 시간 / 총 리드타임

일반적 조직: 5~15%
= 100일 프로젝트 중 실제 작업은 5~15일
```

### 해결책

- **풀 시스템**: 필요할 때만 작업 시작
- **WIP 제한**: 진행 중 작업 수 제한
- **병목 제거**: 가장 느린 단계 집중 개선

---

## 7. 회의 비용 (Meeting Cost)

### 정의

> 회의 자체 + 회의로 인한 생산성 손실 비용

### 계산 공식

```
회의 비용 = 참석자 수 × 시간 × 시급 × 3

왜 ×3?
- 회의 시간 (1)
- 회의 전 준비 (0.5)
- 회의 후 집중력 회복 (0.5)
- 기회비용 (1)

예: 10명 × 1시간 × 5만원 × 3 = 150만원
```

### 숨은 구성 요소

| 비용 | 설명 |
|:---|:---|
| **준비 시간** | 자료 만들기, 읽기 |
| **이동 시간** | 회의실 이동 |
| **전환 비용** | 회의 전후 집중력 손실 |
| **불필요 참석** | "참고용" 참석자 |

### 해결책

- **회의 없는 날**: 특정 요일은 회의 금지
- **15분 기본**: 1시간이 아닌 15분 회의
- **참석자 최소화**: 의사결정자만 참석
- **비동기 대체**: 문서로 대체 가능한 회의 제거

---

## 8. 품질 비용 (Cost of Quality)

### 정의

> 품질을 확보하기 위한 비용 + 품질 미달로 인한 비용

### 구성

```
┌──────────────────────────────────────────────────────────────┐
│  예방 비용 (Prevention)    │  실패 비용 (Failure)            │
├──────────────────────────────────────────────────────────────┤
│  교육, 설계 검토, 프로세스  │  재작업, 고객 불만, 리콜       │
│  투자: 1                   │  손실: 10~100                   │
└──────────────────────────────────────────────────────────────┘
```

### 해결책

- **예방에 투자**: 실패 비용의 1/10로 예방 가능
- **품질 내재화**: 마지막 검수가 아닌 매 단계 품질

---

## 9. 의사결정 지연 비용 (Decision Delay Cost)

### 정의

> 결정을 미루는 동안 발생하는 기회비용과 팀 대기 비용

### 구성 요소

| 비용 | 설명 |
|:---|:---|
| **팀 유휴** | 결정 기다리는 팀원들 |
| **옵션 소멸** | 시간 지나면 선택지 사라짐 |
| **불확실성 유지** | 팀이 방향 없이 표류 |
| **분석 마비** | 더 많은 정보 요구 → 더 지연 |

### 측정

```
지연 비용 = 대기 인원 × 시급 × 지연 시간 + 기회비용
```

### 해결책

- **결정 기한**: 모든 결정에 기한 부여
- **70% Rule**: 70% 확신이면 결정 (Jeff Bezos)
- **가역성 판단**: 되돌릴 수 있으면 빨리 결정

---

## 10. 암묵지 손실 비용 (Tacit Knowledge Loss Cost)

### 정의

> 사람이 떠날 때 문서화되지 않은 지식이 함께 사라지는 비용

### 구성 요소

| 비용 | 설명 |
|:---|:---|
| **재발견 시간** | 이미 알던 것을 다시 알아내기 |
| **실수 반복** | 과거의 실패를 다시 경험 |
| **관계 재구축** | 네트워크, 신뢰 재형성 |
| **온보딩** | 신규 인력 학습 곡선 |

### 측정

```
지식 손실 비용 = 연봉 × 0.5~2.0 (역할에 따라)

핵심 인력 이탈: 연봉의 200%까지 손실
```

### 해결책

- **문서화 문화**: 암묵지 → 형식지 전환
- **페어 작업**: 지식 자연 전파
- **Knowledge Base**: 중앙 집중 지식 저장소
