# Capabilities as Real Options

## Summary
Kogut & Kulatilaka (2001) reframe organizational capabilities as **real option portfolios**: today's capability provides not just current profits but options for future strategic shifts. However, overly strong capabilities create "competency traps" where switching costs become prohibitive.

## Research Question
How should firms conceptualize and value organizational capabilities when the future is uncertain?

## Key Concepts

### Capability as Option Platform
Current capability = {Current profit stream} + {Option to switch} + {Option to expand}

### Competency Trap
When capability becomes so specialized that:
- Switching costs exceed option value
- Cognitive inertia delays exercise decisions
- The option expires worthless

## Main Results

### Bellman Equation Framework
V(K) = max{Ï€(K), E[V(K')] - C}

Optimal policy: Stay with K if current profits exceed expected value of switching minus costs.

### Design Implication
Capabilities should be designed with "built-in" switching optionsâ€”modular, platform-based, transferable.

## Critical Insights

> **Scott Stern Application**: The strategic choice isn't just "which capability to build" but "how many future options does this capability preserve?"

> **Charlie Fine Application**: Operational architecture mattersâ€”modular systems preserve options while integrated systems may trap.

## Methodology
**Approach**: Dynamic optimal control (Bellman equation)
**Data**: Theoretical model with numerical analysis
**Analysis**: State-dependent switching policies
