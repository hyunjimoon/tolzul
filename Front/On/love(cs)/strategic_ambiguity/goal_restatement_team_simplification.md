
## §5. 단순화 이유 (Rationale)

### 5.1 이점

| 이점 | 설명 |
|:---|:---|
| **핸드오프 제거** | G↔J 간 코드 전달/실행 오버헤드 0 |
| **의사결정 속도** | M↔G 직접 소통, 중간 대기 없음 |
| **맥락 유지** | 하나의 AI가 전체 맥락 보유 |
| **K🔴 집중** | 검증 전문가는 검증에만 집중 |

### 5.2 위험 & 완화

| 위험 | 완화 방안 |
|:---|:---|
| G🟠 과부하 | M_統이 우선순위 관리 |
| 검증 약화 | K🔴 D3 스팟 검증으로 품질 확보 |
| J🟢 데이터 전문성 손실 | G🟠이 동일 수준 수행 가능 |
