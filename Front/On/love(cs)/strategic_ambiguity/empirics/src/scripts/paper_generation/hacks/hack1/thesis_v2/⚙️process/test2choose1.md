### 4가지 전략 요약 표

| 전략                                                                                            | **잘 맞는 상황 (Good)**                                                                                                                                                                                                                                                                                                                 | **안 맞는 상황 (Bad)**                                                                                                                                                                                                                                                                                                                                       |
| --------------------------------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **지식재산(IP)**_(Component + Resource Control + Existing Users + Collaboration)_                 | - 수요가 **기존 사용자**이고, 기존 시스템 안에서 **특정 부품/기술(컴포넌트)** 성능 향상이 핵심일 때- **특허·영업비밀 보호가 강하고 집행 가능**한 시장일 때 (예: 생명과학, 제약 라이선스)- “제품”보다 **기술 자체를 라이선스**하는 게 자연스러운 구조일 때- 대기업이 자체 개발보다 **외부 기술 라이선스/공동개발**을 선호하는 관행이 있을 때                                                                                                                       | - 들어가려는 국가/시장에 **특허 보호·집행이 약하거나 거의 없을 때** (IP 무시 문화, 제도 부재)- 해당 기술에서 **특허를 내기 어렵거나, 쉽게 베낄/우회할 수 있을 때** → IP로 캡처가 안 됨- 대기업이 IP를 존중하기보다 **직접 카피·우회 개발**할 가능성이 높은 산업일 때- 빠른 사용자 학습·실험이 중요한데, IP 라이선스 구조 탓에 **사용자와 너무 멀리 떨어져** 학습 속도가 안 나올 때                                                                                                                |
| **Architectural(아키텍처/플랫폼)**_(System + Integrated Resource Control + New Users + Competition)_ | - **마켓플레이스/플랫폼**처럼 **네트워크 효과**가 핵심인 시장일 때- 콜드 스타트 문제를 푸는 **명확한 한쪽 시장의 강력한 가치 제안**이 있을 때 (예: Facebook처럼 먼저 사용자 측을 완전히 모을 수 있을 때)- 한쪽에서 볼륨을 키우고 나면, 나중에 **다른 쪽(광고주·판매자 등)을 붙여 수익화**할 수 있는 구조가 뚜렷할 때- 데이터·표준·핵심 인프라 등 **중요 자원을 어느 정도 통제**하면서도, 그 통제가 생태계에 진짜 효용(질·안전·편의)을 줄 때                                                         | - 양면 시장 어느 쪽에도 **초기 유인을 만들 뚜렷한 방법이 없고**, 콜드 스타트 문제에 대한 설계가 없을 때- 이미 강력한 플랫폼이 있고, 그들과 다른 **네트워크 구축 경로·집중 타깃**을 제시하기 어려운 상황일 때- 창업팀이 **자본·시간 여유가 거의 없을 때**: 아키텍처/플랫폼 전략은 보통 몇 년간 돈을 거의 못 벌고 **볼륨부터 쌓아야** 해서, “부유한 창업자가 택하기 쉬운 전략”이라는 교수 코멘트가 적용되는 상황- 모델이 조금만 검증되면 빅테크가 **copy–paste**하기 쉬운 형태라, 네가 콜드 스타트·학습 비용을 다 내고 나서 시장을 **큰 플레이어에게 통째로 넘길 위험**이 클 때 |
| **Value Chain(밸류체인)**_(Component + Functional Capabilities + Existing Users + Collaboration)_ | - 기존 **밸류체인 안의 한 단계**(제조, 물류, 분석 등)를 **더 싸게·빠르게·정확하게** 만들 수 있을 때- 전체 아키텍처는 그대로 두고, **“효율 개선”만으로도 고객이 바로 가치를 이해**할 수 있을 때- 산업이 **협력 지향적**이고, 대기업이 외부 파트너와 **공정 개선·공동 개발**을 자주 하는 문화일 때- 여러 고객사와 거래해서 **특정 파트너 하나에 과도하게 의존하지 않아도 되는** 구조일 때                                                                                          | - 네가 문제와 솔루션을 잘 정의해서 보여주면, 대기업이 **“고마워” 하고 똑같이 따라 해서** 널 대체할 수 있을 때 (“eat your lunch” 리스크)- 스스로는 “협력자”라고 생각하지만, 실제론 **대기업과 정면 경쟁 포지션**이 되는 구조일 때 (대기업이 너를 경쟁자로 보고 흡수·복제)- 매출이 몇몇 큰 파트너의 **전략·기분에 따라 롤러코스터처럼** “풍년/흉년(feast/famine)”을 오가는 구조일 때- 산업이 점점 **플랫폼/아키텍처 중심**으로 재편되고 있어, 중간 단계 효율화만으론 장기적으로 의미가 줄어들 때 (너는 계속 과거 체인을 최적화만 함)                    |
| **Disruption(파괴적)**_(System + Integrated Capabilities + New Users + Competition)_             | - **새로운 기술**이 새 **S-커브**를 만들 가능성이 크다고 믿을 수 있을 때 (“이 기술이 세상을 먹을 수 있다”)- 그 신기술을 도입하면 기존 기업들의 **조직·규제·수익모델이 꼬여서** 쉽게 못 따라올 때 (incumbent를 knots로 만드는 구조)- **새로운/저가 세그먼트·비소비자**가 있고, 여기에 **통째 시스템**(제품+운영+채널)을 제공할 수 있을 때- 실행만 잘하면 **Uber vs Lyft**처럼 **점유율과 가치 캡처**를 크게 가져올 수 있는 시장일 때- 예: 핀테크처럼 **신기술 + 기존 규제의 비효율**이 스타트업에게 틈을 주는 영역 | - **새 기술이 없고**, 기존 기술·룰이 안정적이며, 대기업이 **시장·규제·채널을 꽉 쥐고** 있을 때 → 사실상 파괴적 기회가 거의 없을 때 (교수 말대로 “그럴 땐 걍 Goldman Sachs 취직”)- 새 기술은 있어도, **대기업이 그냥 가져다 쓸 가능성**이 높을 때 (그들도 새 S-커브를 탈 수 있을 때)- 진입장벽(규제·자본·인허가)이 **대기업에게만 유리**하게 설계돼 있을 때 → 스타트업 disruptive 스토리가 깨짐- 통째 시스템을 만들 **자본·조직 역량도 없는데**, “disruption”이라는 말만 믿고 무리하게 통합 전략을 택할 때                         |
